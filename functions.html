The functions of pyqz¶

pyqz.get_MVphotogrid_metadata(fn)¶

Returns the first 4 lines of a MAPPINGS V file (compiled via the Hawk script).

Parameters:fn – {string} filename (inculding path if required) as string.
Returns:dictionnary of strings
pyqz.refine_MVphotogrid(fn, sampling=2)¶

A function to interpolate the MAPPINGS grid into thiner grid. uses Akyma splines (1-D) in the Q/Z planes (one axis at a time).

Parameters:
  • fn – {string} filename (inculding path if required) as string.
  • sampling – {int; default = 2} the refining factor, i.e. number of interpolated points between each native grid node.
pyqz.get_grid(ratios, coeffs=[[1, 0], [0, 1]], Pk=5.0, kappa=inf, model='sph', resampled=True)¶

Returns a given line ratio diagnostic grid generated using MAPPINGS for a given kappa value.

Parameters:
  • ratios – {string} The line ratios involved in the grid, e.g. ‘[NII]/[SII]+;[OIII]/Hb’
  • coeffs – {list; length = 2, default = [[1,0],[0,1]]} The different coefficients with which to mix the different line ratios. The size of each list elements must be equal to the number of line ratios involved.
  • Pk – {float;default = 5.0} MAPPINGS model pressure. Value must match an existing reference grid file !
  • kappa – {float; default = np.inf} The kappa value. Value must match an existing reference grid file !
  • model – {string; default = ‘sph’} spherical (‘sph’) or plane-parallel (‘pp’) HII regions. Value must match an existing reference grid file !
  • resampled – {bool; default = True} Use a resampled grid ?
Returns:

[grid,grid_cols,metadata] grid {numpy array}: the diagnostic grid as a numpy array. grid_cols {list}: labels of each column inside grid. metadata {list}: basic info from the MAPPINGS simulations.

pyqz.interp_qz(qz, data, ratios, coeffs=[[1, 0], [0, 1]], Pk=5.0, kappa=inf, model='sph', resampled=True, method='cubic', show_plot=False, n_plot=False, save_plot=False)¶

The core function of pyqz.

Returns the ‘q’ or ‘z’ value for a given set of line ratios based on a given 2-D diagnostic grid.

Parameters:
  • qz – {string} Which estimate to return, ‘LogQ’, ‘Tot[O]+12’ or ‘gas[O]+12’
  • data – {list of numpy array} Array of the line ratio values. One array per line ratio of size NxM for NxM values).
  • ratios – {string} The line ratios involved in the grid, e.g. ‘[NII]/[SII]+;[OIII]/Hb’
  • coeffs – {list; length = 2, default = [[1,0],[0,1]]} The different coefficients with which to mix the different line ratios. The size of each list elements must be equal to the number of line ratios involved.
  • Pk – {float;default = 5.0} MAPPINGS model pressure. Value must match an existing reference grid file !
  • kappa – {float; default = np.inf} The kappa value. Value must match an existing reference grid file !
  • model – {string; default = ‘sph’} spherical (‘sph’) or plane-parallel (‘pp’) HII regions. Value must match an existing reference grid file !
  • resampled – {bool; default = True} Use a resampled grid ?
  • method – {string; default = ‘cubic’} ‘linear’ or ‘cubic’ interpolation method of scipy.interpolate.griddata.
  • show_plot – {bool; default = False}
  • n_plot – {int; default = False} defines the plot window number
  • save_plot – {string; default = False} ‘path+name+format’ to export the Figure to.
Returns:

{numpy array of NxM}
pyqz.get_global_qz(data, data_cols, which_grids, ids=False, qzs=['LogQ', 'Tot[O]+12'], Pk=5.0, kappa=inf, model='sph', resampled=True, error_pdf='normal', srs=400, flag_level=2.0, KDE_method='gauss', show_plot=False, save_plot=False, plot_loc='./', plot_fmt='png')¶

Derives the LogQ and [O]+12 values for a given set of line fluxes based on a given set of diagnostic grids.

This function only uses “valid” grids defined in pyqz_metadata.py.

For each set of line ratios, this function combines the estimates from invdividual diagnostics into a combined best estimate of LogQ and [O]+12. Observational errors are propagated via the joint probability density function, reconstructed via a Kernel Density Estimation.

Parameters:
  • data – {numpy array of size NxMx2} N sets of M line fluxes and errors. An error =-1 implies the associated line flux is an upper limit.
  • data_cols – {list of Mx2 strings} Data content, e.g. [‘[NII]+,stdHa’,...] Syntax MUST match the MAPPINGS + pyqz conventions !
  • which_grids – {list of strings} list of the model grids to use for the estimations, e.g. [‘[NII]+/Ha;[OII]/Hb’,...]
  • ids – {list; default = False} an optional length-N list of numbers or string to identify each data point (e.g. spaxel number, source ID, etc ...)
  • qzs – {list; default = [‘LogQ’,’Tot[O]+12’]} list of Q/Z values to compute
  • Pk – {float;default = 5.0} MAPPINGS model pressure. Value must match an existing reference grid file !
  • kappa – {float; default = np.inf} The kappa value. Value must match an existing reference grid file !
  • model – {string; default = ‘sph’} spherical (‘sph’) or plane-parallel (‘pp’) HII regions. Value must match an existing reference grid file !
  • resampled – {bool; default = True} Use a resampled grid ?
  • error_pdf – {string; default = ‘normal’} The shape of the error function for the line fluxes. Currently, only ‘normal’ is supported.
  • srs – {int; default = 400} The number of random line fluxes generated to discretize (and reconstruct) the joint probability function.
  • flag_level – {float; default = 2} A ‘flag’ is raised (in the output file) when the direct q/z estimate and the KDE q/z estimate (q_rs and z_rs) are offset by more than flag_level * standard_deviation. Might indicate trouble. A flag is always raised when a point is outside all of the grids (8), or when the KDE PDF is multipeaked (9).
  • KDE_method – {string; default = ‘gauss’} Whether to use scipy.stats.gaussian_kde (‘gauss’) or sm.nonparametric.KDEMultivariate (‘multiv’) to perform the 2-D Kernel Density Estimation
  • show_plot – {string/bool; default = False} Show all plots (True), none (False), only the grids (‘grids’) or the KDE maps (‘KDE’).
  • save_plot – {string/bool; default = False} Save all plots (True), none (False), only the grids (‘grids’), only the KDE maps (‘KDE_all’) or just the KDE maps raising a flag (‘KDE_flag’).
  • plot_loc – {string; default = ‘./’} Location where figures are saved.
  • plot_fmt – {string; default = ‘png’} ‘eps’, ‘pdf’, or ‘png’.
Returns:

[P,r], where P contains all the estimated Log(Q) and [O]+12 values, and r contains the columns name

The get_global_qz-‘from a file’ function. Gets the line ratios from a csv file, and not directly as Python arrays.

Save to file the log(q) and 12+log(O/H) values for a given set of line fluxes based on a given set of diagnostic grids. Requires a specific input file, and can batch process several measurements automatically.

Parameters:
  • fn – {string} The path+filename of the input file, in CSV format
  • which_grids –
    {list of strings}
    list of the model grids to use for the estimations, e.g. [‘[NII]+/Ha;[OII]/Hb’,...]
    param qzs:{list; default = [‘LogQ’,’Tot[O]+12’]} list of Q/Z values to compute
  • Pk – {float;default = 5.0} MAPPINGS model pressure. Value must match an existing reference grid file !
  • kappa – {float; default = np.inf} The kappa value. Value must match an existing reference grid file !
  • model – {string; default = ‘sph’} spherical (‘sph’) or plane-parallel (‘pp’) HII regions. Value must match an existing reference grid file !
  • resampled – {bool; default = True} Use a resampled grid ?
  • error_pdf – {string; default = ‘normal’} The shape of the error function for the line fluxes. Currently, only ‘normal’ is supported.
  • srs – {int; default = 400} The number of random line fluxes generated to discretize (and reconstruct) the joint probability function.
  • flag_level – {float; default = 2} A ‘flag’ is raised (in the output file) when the direct q/z estimate and the KDE q/z estimate (q_rs and z_rs) are offset by more than flag_level * standard_deviation. Might indicate trouble. A flag is always raised when a point is outside all of the grids (8), or when the KDE PDF is multipeaked (9).
  • KDE_method – {string; default = ‘gauss’} Whether to use scipy.stats.gaussian_kde (‘gauss’) or sm.nonparametric.KDEMultivariate (‘multiv’) to perform the 2-D Kernel Density Estimation
  • decimals – {int; default = 5} Number of decimals to print in the final CSV file.
  • missing_values – {string; default = ‘$$$’} Symbol used to mark missing values.
  • suffix_out – {string; default = ‘_out’} String to add to the input filename. Warning: ‘’ will overwrite the input file !!!
  • show_plot – {string/bool; default = False} Show all plots (True), none (False), only the grids (‘grids’) or the KDE maps (‘KDE’).
  • save_plot – {string/bool; default = False} Save all plots (True), none (False), only the grids (‘grids’), only the KDE maps (‘KDE_all’) or just the KDE maps raising a flag (‘KDE_flag’).
  • plot_loc – {string; default = ‘./’} Location where figures are saved.
